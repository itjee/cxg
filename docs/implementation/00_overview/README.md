# 구현 문서 (Implementation Documents)

이 폴더는 프로젝트 개발 과정에서 생성된 구현 문서들을 시스템별로 보관합니다.

## 폴더 구조

```
implementation/
├── backend-api/    # 백엔드 API 구현 이력
├── manager-web/    # 매니저 웹 구현 이력
├── tenants-web/    # 테넌트 웹 구현 이력
└── shared/         # 공통/전체 시스템 구현 이력
```

## 파일명 규칙

모든 구현 문서는 다음 형식으로 저장됩니다:

```
{변경내용_한글}_{yyyymmddhhmmss}.md

예시:
- 인증플로우_수정_20251017210930.md
- 테마시스템_스크롤바_구현_20251017211208.md
- API_라우트_참조_20251017155434.md
- 사용자관리_구현_20251015213022.md
```

**중요**: 파일명은 **한글**로 작성하며 띄어쓰기는 언더스코어(_)로 구분합니다

**타임스탬프 형식**: `yyyymmddhhmmss` (한국시간 KST 기준)
- yyyy: 연도 (4자리)
- mm: 월 (2자리)
- dd: 일 (2자리)
- hh: 시 (24시간, 2자리)
- mm: 분 (2자리)
- ss: 초 (2자리)

## 문서 생성 시 주의사항

### 1. 타임스탬프 자동 생성
```bash
# Bash
TIMESTAMP=$(date +"%Y%m%d%H%M%S")
touch "feature_name_${TIMESTAMP}.md"

# Python
from datetime import datetime
timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
filename = f"feature_name_{timestamp}.md"
```

### 2. 문서 제목
- 파일명은 한글 사용 (띄어쓰기는 언더스코어 _)
- 문서 내 제목은 명확하고 구체적으로 작성
- 날짜와 작성자 명시 권장

### 3. 문서 구조

**모든 문서는 한글로 작성합니다**

```markdown
# [기능/변경사항 제목]

**날짜**: YYYY-MM-DD HH:MM:SS KST
**작성자**: [작성자명 또는 시스템]
**유형**: [기능 구현 | 버그 수정 | 리팩토링 | 개선]
**컴포넌트**: [영향받은 컴포넌트 목록]

## 개요
변경된 내용과 이유에 대한 간단한 설명

## 식별된 문제점 (해당시)
이 변경을 하게 된 문제점

## 변경 내용
수정 사항에 대한 상세 설명

### 코드 예시
수정 전/후 비교를 포함한 관련 코드 스니펫

## 변경된 파일 목록
이 변경으로 영향받은 모든 파일 목록

## 테스트
변경사항을 테스트하거나 검증하는 방법

## 향후 개선사항 (선택)
잠재적인 개선사항이나 처리할 기술 부채

## 참고사항
추가 컨텍스트나 고려사항
```
