## 주요 변경사항

### 1. StatsCard 공통 컴포넌트 생성

**파일**: `apps/tenants-web/src/components/stats-card.tsx`

manager-web의 디자인 패턴을 그대로 적용한 재사용 가능한 컴포넌트 생성:

```typescript
interface StatsCardProps {
  title: string;
  value: string | number;
  description?: string;
  icon: ReactElement;
  iconColor: string;
}
```

#### 주요 디자인 요소

**좌측 컬러 보더**
- 각 카드의 테마 컬러로 4px 두께의 좌측 보더 추가
- 시각적 구분 및 브랜드 아이덴티티 강화
```tsx
className="border-l-4"
style={{ borderLeftColor: iconColor }}
```

**우측 배경 아이콘**
- 위치: 카드 우측, 수직/수평 중앙 정렬
- 크기: 80px × 80px
- 효과:
  - 15도 우측 회전 (`transform: rotate(15deg)`)
  - 20% 투명도 (`opacity: 0.2`)
  - 카드 경계를 살짝 넘어가도록 배치 (`-mr-3`)
  - hover 시 1.1배 확대 효과
- React `cloneElement`를 사용하여 동적으로 아이콘 크기 오버라이드

**다층 그라디에이션 효과**

1. 메인 대각선 그라디에이션:
```tsx
background: `linear-gradient(135deg, ${iconColor} 0%, transparent 60%)`
opacity: 0.04 (hover: 0.08)
```

2. 보조 방사형 그라디에이션:
```tsx
background: `radial-gradient(circle at top right, ${iconColor} 0%, transparent 50%)`
opacity: 0.02 (hover: 0.05)
```

3. 상단 하이라이트 효과:
```tsx
background: `linear-gradient(90deg, transparent, ${iconColor}, transparent)`
height: 1px
opacity: 0.3 (hover: 0.5)
```

**레이아웃 구조**
- Flexbox 레이아웃: `flex-row`
- 좌측: 콘텐츠 영역 (제목, 값) - `flex-1`로 확장
- 우측: 아이콘 영역 - 100px 고정 너비
- 카드 높이: 120px 고정

**호버 인터랙션**
- 그림자: `hover:shadow-xl hover:shadow-primary/10`
- 이동: `hover:-translate-y-1` (4px 상승)
- 지속시간: `duration-300`
- 보더: `hover:border-primary/30`
- 모든 그라디에이션 레이어 투명도 증가
- 아이콘 1.1배 확대

## 기술적 세부사항

### React 아이콘 크기 동적 변경
```typescript
import { cloneElement, isValidElement } from "react";

{isValidElement(icon)
  ? cloneElement(icon, { className: "h-[80px] w-[80px]" } as any)
  : icon
}
```
- 부모 컴포넌트에서 전달된 아이콘의 크기를 80px로 강제 변경
- `cloneElement`를 사용하여 props 오버라이드
- 타입 안전성을 위해 `isValidElement` 체크 추가

### CSS 다층 그라디에이션 레이어링
```tsx
{/* 레이어 1: 대각선 그라디에이션 */}
<div className="absolute inset-0 opacity-[0.04]" />

{/* 레이어 2: 방사형 그라디에이션 */}
<div className="absolute inset-0 opacity-[0.02]" />

{/* 레이어 3: 상단 하이라이트 */}
<div className="absolute top-0 h-[1px]" />
```
- 여러 개의 `absolute` 레이어를 겹쳐서 복잡한 시각 효과 생성
- 각 레이어마다 다른 투명도와 전환 효과 적용

### Tailwind CSS 커스텀 값 활용
- `h-[120px]`: 카드 정확한 높이
- `h-[80px] w-[80px]`: 아이콘 크기
- `opacity-[0.04]`: 세밀한 투명도 조절
- `-mr-3`: 네거티브 마진으로 아이콘을 컨테이너 밖으로
- `pt-[15px] pb-[5px]`: 정확한 패딩 값

## 테스트 방법

### 개발 서버 실행
```bash
cd apps/tenants-web
pnpm dev
```

### 시각적 검증
각 기준정보 페이지 접속 후 확인:
1. 좌측 컬러 보더가 표시되는지
2. 우측 아이콘이 15도 기울어져 있는지
3. 배경 그라디에이션 효과 확인
4. 값(value)이 크고 명확하게 표시되는지

### 인터랙션 테스트
각 카드에 마우스 호버 시:
1. 카드가 위로 4px 이동
2. 그림자 강화
3. 아이콘 확대 (1.1배)
4. 그라디에이션 투명도 증가
5. 모든 전환이 300ms 동안 부드럽게 진행

### 반응형 테스트
다양한 화면 크기에서 테스트:
- Mobile: 카드가 1열로 쌓임
- Tablet: 카드가 2x2 그리드로 배치
- Desktop: 카드가 1x4 그리드로 배치

## UX 개선 효과

### 시각적 계층 구조
1. **좌측 컬러 보더**: 카드 카테고리를 색상으로 즉시 식별
2. **큰 값(Value)**: 가장 중요한 정보를 강조 (2xl font)
3. **배경 아이콘**: 장식적 요소이면서 카드 유형 암시
4. **제목**: 보조 정보로 작은 글씨 (text-sm)

### 사용자 경험 향상
- **정보 스캔 속도 향상**: 컬러 코딩과 큰 숫자로 빠른 정보 파악
- **인터랙티브 피드백**: 호버 효과로 클릭 가능성 암시
- **시각적 즐거움**: 부드러운 애니메이션과 그라디에이션으로 프리미엄 느낌
- **일관성**: 모든 기준정보 페이지에 동일한 패턴 적용

### 접근성 고려
- 색상만으로 정보를 전달하지 않음 (텍스트 레이블 병기)
- 충분한 색상 대비 유지
- 호버 외에도 정보가 명확히 표시됨

## 향후 개선사항

### 다크 모드 최적화
- 현재 그라디에이션 투명도는 라이트/다크 모드 공통
- 필요 시 다크 모드에서 더 강한 효과 적용 고려:
```tsx

```

### 애니메이션 고도화
- Framer Motion 도입 시 더 복잡한 호버 효과 가능
- 아이콘 회전 애니메이션, 스프링 효과 등

### 데이터 시각화 강화
- trend 정보가 있는 경우 작은 차트나 스파크라인 추가 고려
- 전월 대비 증감 표시 (+5%, -3% 등)

### Skeleton Loading
- 데이터 로딩 중 Skeleton UI 표시
- 카드 구조는 유지하면서 콘텐츠만 Skeleton으로 대체
