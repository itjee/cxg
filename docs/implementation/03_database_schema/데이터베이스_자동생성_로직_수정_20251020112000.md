# 작업: 데이터베이스 자동 생성 로직 수정

## 요청
- `backend-api` 구동 시 데이터베이스 테이블을 자동으로 생성하는 로직을 제거.
- 생성 로직을 데이터베이스 연결 및 존재 여부를 확인하는 로직으로 변경 요청.

## 진행
1.  `apps/backend-api/src/main.py` 파일을 분석하여 앱 시작 시 `init_db` 함수가 호출됨을 확인했습니다.
2.  `apps/backend-api/src/core/database.py` 파일을 분석하여 `init_db` 함수 내의 `Base.metadata.create_all` 호출 로직을 식별했습니다.
3.  `init_db` 함수를 `SELECT 1` 쿼리를 통해 데이터베이스 연결만 테스트하는 코드로 교체했습니다.
    - 연결 실패 시, 어떤 DB(테넌트/관리자)에서 문제가 발생했는지와 연결 정보를 명확히 출력하도록 오류 메시지를 개선했습니다.
4.  새로운 `init_db` 함수에서 사용하는 `text` 함수를 import 하기 위해 `from sqlalchemy import text` 구문을 추가했습니다.

## 결과
- `apps/backend-api/src/core/database.py` 파일의 `init_db` 함수가 성공적으로 수정되었습니다.
- 이제 `backend-api`는 시작 시 테이블을 자동으로 생성하지 않고, 데이터베이스 연결 상태만 확인합니다.
