## 디자인 원칙

### 1. 중립성 (Neutrality)
- **배경과 UI**: Neutral 그레이스케일 사용
- **콘텐츠 집중**: 색상을 최소화하여 정보에 집중
- **전문성**: 엔터프라이즈 SaaS에 적합한 차분한 색상

### 2. 명확한 시각적 계층 (Clear Hierarchy)
- **Primary**: Violet으로 주요 액션 강조
- **Neutral**: 배경과 텍스트의 명확한 대비
- **Semantic**: 초록(성공), 빨강(위험), 주황(경고)만 사용

### 3. 접근성 (Accessibility)
- **대비율**: WCAG AAA 기준 충족
- **포커스**: Violet ring으로 명확한 포커스 표시
- **색맹 대응**: 중요 정보를 색상만으로 전달하지 않음

### 4. 일관성 (Consistency)
- **두 앱 통일**: Manager-web과 Tenants-web 동일한 컬러 시스템
- **테마 전환**: 라이트/다크 모드 간 부드러운 전환
- **브랜드 정체성**: Violet primary로 브랜드 통일

## 색상 사용 가이드

### ✅ 해야 할 것 (DO)

1. **Semantic 토큰 사용**
   ```tsx

   className="bg-primary text-primary-foreground"
   className="bg-card text-card-foreground"
   className="border-border"
   ```

2. **상태 표시는 색상 유지**
   ```tsx

   ACTIVE: "bg-emerald-500/10 text-emerald-700"
   SUSPENDED: "bg-destructive/10 text-destructive"
   ```

3. **Primary는 중요한 액션에만**
   ```tsx

   <Button className="bg-primary">회원가입</Button>
   ```

### ❌ 하지 말아야 할 것 (DON'T)

1. **하드코딩된 색상 사용 금지**
   ```tsx
   // ❌ Bad
   className="bg-gray-100 text-gray-900"

   // ✅ Good
   className="bg-muted text-foreground"
   ```

2. **과도한 색상 사용**
   ```tsx
   // ❌ Bad - 너무 많은 색상
   <div className="bg-blue-500">
     <div className="bg-green-500">
       <div className="bg-purple-500">

   // ✅ Good - neutral 기반, primary만 강조
   <div className="bg-card">
     <div className="bg-muted">
       <Button className="bg-primary">
   ```

3. **Primary를 배경으로 과도하게 사용**
   ```tsx
   // ❌ Bad - primary 남용
   <div className="bg-primary">...</div>

   // ✅ Good - accent로 제한적 사용
   <Button className="bg-primary">저장</Button>
   ```

## 향후 개선 사항

### 1. 다크 모드 변형
현재는 하나의 다크 모드만 제공하지만, 향후 다양한 변형을 추가할 수 있습니다:

```css
.dark-soft {
  --background: var(--neutral-800);  /* 덜 어두운 배경 */
}

.dark-true {
  --background: oklch(0% 0 0);      /* 순수 검은색 배경 */
}
```

### 2. Accent 색상 추가
Violet 외에 추가 accent 색상 정의:

```css
--accent-blue: oklch(59% 0.20 240);
--accent-green: oklch(71.5% 0.177 145);
--accent-orange: oklch(78.9% 0.171 70);
```

색상 전환 애니메이션 개선:

```css
@media (prefers-reduced-motion: no-preference) {
```

### 4. 컴포넌트 레벨 테마

## 참고 사항

### OKLCH vs RGB/HSL

**OKLCH의 장점**:
1. **지각적 균일성**: 인간 시각과 일치하는 밝기
2. **예측 가능성**: L값이 실제 밝기를 정확히 반영
3. **넓은 색역**: P3 디스플레이의 모든 색상 표현
4. **접근성**: 대비율 계산이 정확함

**OKLCH 문법**:
```css
oklch(L% C H)
     ↑  ↑ ↑
     밝기 채도 색조
```

### 색상 네이밍 규칙

- **Neutral**: 무채색 (Chroma = 0)
- **Violet**: Hue ≈ 293-295도
- **숫자**: 50-950 (밝음 → 어두움)
- **Semantic**: primary, secondary, muted, accent, destructive

### 색상 도구

- **Contrast Checker**: https://www.siegemedia.com/contrast-ratio
- **OKLCH Picker**: https://oklch.com
- **Color Palette**: https://www.radix-ui.com/colors
