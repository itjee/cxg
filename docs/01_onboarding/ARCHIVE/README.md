# Documentation

This directory contains all project documentation organized by purpose.

## Directory Structure

```
docs/
├── guides/          # Project guides and development documentation (01-10 numbered guides)
├── generated/       # Auto-generated documentation (API docs, models, schemas)
├── reports/         # Project reports and completion documents
├── auth/            # Authentication and authorization guides
└── sample/          # Sample files and examples
```

### Generated Documentation
- Located in `generated/` directory
- Can be regenerated using project scripts
- Should not be manually edited

### Reports
- Implementation and completion reports in `reports/` directory
- Includes Korean and English versions
- Historical record of project progress

### Authentication
- User guides for authentication features in `auth/` directory
- Login and registration procedures

## Maintenance

- **guides/**: Manually maintained, keep up-to-date with project changes
- **generated/**: Auto-generated, can be safely deleted and regenerated
- **reports/**: Archive-only, add new reports with date prefixes
- **auth/**: User-facing documentation, update when features change
